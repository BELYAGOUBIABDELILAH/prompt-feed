# Explain Unfamiliar Code or MR

**Category:** Coding & Development  
**Source:** GitLab Duo

## Prompt

> Help me quickly understand this unfamiliar code or MR:
> 
> [PASTE CODE or MR LINK]
> 
> I need to understand this because: [reviewing it / modifying it / debugging it / onboarding]
> 
> ## TL;DR (30 seconds)
> In 3 sentences: what is this, what does it do, and why does it exist?
> 
> ## Architecture Context
> - Which layer of the system is this in? (data / business logic / API / UI)
> - What other components does this interact with?
> - Who created this and when? (if visible)
> 
> ## Code Flow
> Walk me through the execution path from entry point to exit, focusing on the main scenario.
> 
> ## The Tricky Parts
> Highlight the 2–3 most confusing sections and explain them clearly.
> 
> ## Before I Modify This
> What should I know before making changes?
> - What could break if I change X?
> - Are there tests that cover this?
> - Are there related files I need to update together?
