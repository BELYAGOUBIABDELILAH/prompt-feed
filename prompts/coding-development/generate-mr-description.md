# Generate MR Description

**Category:** Coding & Development  
**Source:** GitLab Duo

## Prompt

> Generate a comprehensive merge request description based on these changes:
> 
> Branch name: [BRANCH NAME]
> Changed files: [LIST FILES]
> Commit messages: [LIST COMMITS]
> Related issue: [ISSUE # or URL]
> 
> ## MR Description Template to Fill
> 
> ### What
> [1–2 sentences: what does this MR do?]
> 
> ### Why
> [1–2 sentences: what problem does it solve? Link to issue.]
> 
> ### How
> [Bullet list of the key implementation decisions made]
> 
> ### Testing Done
> - [ ] Unit tests: [describe what's covered]
> - [ ] Integration tests: [describe what's covered]
> - [ ] Manual testing steps:
>   1. [Step 1]
>   2. [Step 2]
>   3. Expected result: [...]
> 
> ### Screenshots / Recordings
> [If UI changes: before/after screenshots]
> 
> ### Checklist
> - [ ] Tests added/updated
> - [ ] Documentation updated
> - [ ] No breaking changes OR breaking changes documented
> - [ ] Feature flag added if needed
> - [ ] Performance impact considered
> 
> ### Related
> Closes #[ISSUE]
