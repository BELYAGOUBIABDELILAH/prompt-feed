# Implement Design Pattern

**Category:** Coding & Development  
**Source:** GitLab Duo

## Prompt

> Implement the following design pattern:
> 
> Pattern: [PATTERN NAME — e.g., Repository, Observer, Factory, Strategy, Command]
> Language: [SPECIFY]
> Context: [what problem I'm trying to solve]
> 
> ## Pattern Explanation
> Before implementation:
> 1. What problem does this pattern solve?
> 2. When should I use it? When should I avoid it?
> 3. What are the participants (classes/interfaces involved)?
> 
> ## Complete Implementation
> 
> ### Interfaces / Abstract Classes
> [Define the contracts]
> 
> ### Concrete Implementations
> [Implement 2–3 concrete examples]
> 
> ### Client Code
> [Show how to use the pattern from calling code]
> 
> ## Real-World Example
> Apply this pattern to my specific context: [DESCRIBE YOUR USE CASE]
> Show the complete working implementation for my use case.
> 
> ## Tests
> Write tests that verify the pattern works correctly:
> - Test the abstraction (not implementation details)
> - Test that implementations are interchangeable
> - Test edge cases
> 
> ## Anti-Patterns to Avoid
> List 2–3 common mistakes when implementing this pattern.
