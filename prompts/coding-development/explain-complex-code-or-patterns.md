# Explain Complex Code or Patterns

**Category:** Coding & Development  
**Source:** GitLab Duo

## Prompt

> Explain this complex code in plain language:
> 
> [PASTE CODE]
> 
> ## High-Level Summary
> In 2–3 sentences: what does this code do and why does it exist?
> 
> ## Step-by-Step Walkthrough
> Walk through the logic line by line (or block by block), explaining:
> - What each significant step does
> - Why it's done this way (not just what)
> - Any assumptions or preconditions required
> 
> ## Key Concepts
> Explain any non-obvious patterns, algorithms, or language features used:
> - Design patterns (e.g., "this is the Observer pattern because...")
> - Language-specific tricks (e.g., "this destructuring assignment means...")
> - Domain concepts (e.g., "MRR is calculated this way because...")
> 
> ## Potential Issues
> Point out any parts of the code that:
> - Could behave unexpectedly under certain inputs
> - May be difficult to maintain
> - Contain known inefficiencies
> 
> ## Analogy
> Give a real-world analogy that helps a non-expert understand what this code does.
