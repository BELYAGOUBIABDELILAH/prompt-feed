# Improve UI Accessibility

**Category:** Image & Design  
**Source:** GitLab Duo

## Prompt

> Review and improve the accessibility of this UI component/page:
> 
> [PASTE HTML / JSX / Component Code]
> 
> ## Accessibility Audit
> 
> ### Perceivable
> - [ ] All images have meaningful alt text (or alt="" if decorative)
> - [ ] Color is not the only way information is conveyed
> - [ ] Sufficient color contrast ratio (4.5:1 for normal text, 3:1 for large text)
> - [ ] Text can be resized up to 200% without loss of content
> - [ ] Videos/audio have captions or transcripts
> 
> ### Operable
> - [ ] All functionality accessible via keyboard
> - [ ] No keyboard traps
> - [ ] Skip navigation link present
> - [ ] Focus indicators visible
> - [ ] No content that flashes more than 3 times per second
> 
> ### Understandable
> - [ ] Language of page specified (lang attribute)
> - [ ] Form inputs have associated labels
> - [ ] Error messages are clear and specific
> - [ ] Instructions don't rely solely on shape/location/color
> 
> ### Robust
> - [ ] Valid HTML (no duplicate IDs, proper nesting)
> - [ ] ARIA roles and attributes used correctly
> - [ ] Works with screen readers (NVDA, VoiceOver)
> 
> ## Fixed Code
> Provide the fully accessible version with:
> - All ARIA attributes added
> - Keyboard navigation implemented
> - Focus management corrected
> - Screen reader announcements added
